## 七、EMU与字号对照

python-docx 使用 EMU（English Metric Unit）作为尺寸单位：

| EMU | 字号(中文) | pt | 用途 |
|-----|-----------|-----|------|
| 457200 | 初号 | 36 | 封面标题 |
| 279400 | 二号 | 22 | H1 |
| 203200 | 二号 | 16 | H1(新规范) / 封面副标题 |
| 190500 | 小三 | 15 | H2 |
| 177800 | 四号 | 14 | H3 / 封面信息 |
| 152400 | 小四 | 12 | 正文 |
| 139700 | 五号 | 11 | 表格 |
| 127000 | 小五 | 10 | 小号文字 |

转换公式：px = EMU / 12700

## 八、内容排版模式

### 【招标要求】+【我方响应】模式

```
【招标要求】+ 宋体正文 + 引用招标文件原文

【我方响应】+ 宋体正文 + 详细方案内容
```

- "【招标要求】"和"【我方响应】"作为标签加粗
- 两者之间不留空行
- 不同要求的响应之间留一个空行

### 列表编号

正文中的有序列表使用"（1）（2）（3）..."格式，不使用自动编号。

## 九、emoji和特殊字符

- 不使用emoji
- 乘号使用"×"（如 7×24）
- 连接号使用"—"（如 2026年—2027年）
- 间隔号使用"·"（如 7·10小时）

## 十、重要：eastAsia 字体属性

**python-docx 的已知缺陷**：`run.font.name = '宋体'` 只设置 `w:ascii` 和 `w:hAnsi`
属性，不会设置 `w:eastAsia` 属性。在 OOXML 规范中，中文字体必须通过
`w:eastAsia` 指定。缺少此属性时：

- **Microsoft Word 桌面版**会用系统默认字体替换（如等线），导致字体显示异常
- **VS Code / Web 渲染器**有更好的 Unicode 回退，可能不受影响

**解决方法**：生成 docx 时必须同时设置 `w:eastAsia` 属性。可以在设置字体后
通过 XML 操作补丁：

```python
from docx.oxml.ns import qn
from docx.oxml import OxmlElement

def set_run_font(run, font_name):
    run.font.name = font_name
    rPr = run._element.get_or_add_rPr()
    rFonts = rPr.find(qn('w:rFonts'))
    if rFonts is None:
        rFonts = OxmlElement('w:rFonts')
        rPr.insert(0, rFonts)
    rFonts.set(qn('w:eastAsia'), font_name)
```

参见 `scripts/generate_bid_template.py` 中的 `set_run_font()` 完整实现。
